# 《食藏录》页面结构（当前实现）

## 1. 微信小程序路由

`miniprogram/miniprogram/app.json` 当前注册：

| 页面 | 路由 | 当前职责 |
| --- | --- | --- |
| 登录 | `pages/login/index` | `wx.login` + 后端微信登录，失败可重试 |
| 首页 | `pages/home/index` | 主打卡入口、搜索/手动入口、状态统计、最近记录、今日手账 |
| 店铺搜索 | `pages/store-search/index` | 关键词防抖、城市降级、用户主动定位、腾讯 POI 结果 |
| 店铺详情 | `pages/store-detail/index` | 私人店铺快照与现有记录入口 |
| 打卡编辑 | `pages/check-in-editor/index` | 手动建店、创建/编辑、草稿、校验、重复提交保护与离开提醒 |
| 记录列表 | `pages/records/index` | 状态筛选、搜索、分页、加载/空/失败状态 |
| 记录详情 | `pages/record-detail/index` | 详情、收藏反馈、编辑与删除确认 |
| 今日手账 | `pages/daily-journal/index` | 当日 VISITED 记录 Canvas 2D 预览与保存 |
| 我的 | `pages/profile/index` | `GET /users/me`、记录入口、退出会话 |

首页与搜索、编辑属于主流程。当前未强行分包，因为现有包体仍小；引入地图 SDK、上传与多模板素材前再根据开发者工具包体报告拆分，避免破坏已上线链接。

## 2. 核心跳转

```text
登录
  -> 首页
      -> 附近餐厅/搜索 -> POI -> 新建打卡
      -> 手动添加 -> 定位或经纬度 -> 新建打卡
      -> 状态清单/最近记录 -> 记录列表/详情
      -> 今日手账 -> Canvas 预览 -> 保存相册
      -> 我的 -> 全部记录/退出登录

记录详情 -> 店铺详情 / 编辑 / 收藏 / 删除
```

不存在资源或登录失效时不展示其他用户内容；HTTP 层只允许一次 refresh 并在失败后回登录页。

## 3. 页面状态

已覆盖：首次加载、加载中、空数据、请求失败、重试、登录失效、定位拒绝/失败、保存中、保存成功、删除确认、重复点击、分页加载与没有更多数据。

图片上传进度、图片失败重试、账号注销与网络离线横幅随对应 P1 能力一起实现，不保留无后端支撑的假按钮。

## 4. 定位策略

- 启动与进入搜索页时不申请定位。
- 只有点击“使用当前位置”或手动建店中的定位按钮才调用 `getLocation`。
- 拒绝后保留城市、关键词、手动地址和经纬度输入；可通过设置页重新授权。
- 搜索结果明确来自腾讯位置服务；用户自建店铺只存在于自己的数据库空间。

## 5. Flutter

Flutter 当前入口展示食藏录品牌和真实后端健康状态，API URL 通过 `FLUTTER_API_BASE_URL` 构建配置注入，不在 UI 暴露。微信移动端登录与完整记录流程仍为 P1，未展示不可用的业务按钮。

## 6. 后续页面

P1 根据真实接口补充图片上传、收藏独立列表、地图、统计、标签管理、账号隐私/注销和 Flutter 登录/记录页。年度报告、AI 推荐、社交、支付与商家功能不进入 MVP。
