# 《食迹》页面结构与交互说明（MVP）

## 1. 设计目标

小程序与 App 使用同一套信息架构和业务术语，降低学习与测试成本；具体控件遵循各平台习惯，不追求像素级一致。界面只呈现用户自己的记录，不出现社区、榜单、商家运营或付费入口。

## 2. 全局导航

登录后使用四个一级入口：

| 一级入口 | Route | 核心任务 |
| --- | --- | --- |
| 首页 | `/home` | 快速新增、查看摘要与最近记录 |
| 清单 | `/records` | 管理想去、已打卡和黑名单 |
| 地图 | `/map` | 按空间查看个人美食足迹 |
| 我的 | `/profile` | 统计、标签与应用设置 |

小程序使用原生 `tabBar`，Flutter 使用底部 `NavigationBar`。从二级页面返回时保留一级页面的筛选、滚动位置和地图视口；退出登录后重置所有导航状态。

## 3. 页面总览

```text
启动
├── 登录 /login
└── 主框架
    ├── 首页 /home
    │   ├── 搜索店铺 /stores/search
    │   │   └── 手动添加店铺 /stores/manual
    │   └── 新建记录 /records/create
    ├── 清单 /records
    │   ├── 筛选 /records/filter
    │   ├── 记录详情 /records/:recordId
    │   │   ├── 编辑记录 /records/:recordId/edit
    │   │   └── 图片预览 /images/preview
    │   └── 新建记录 /records/create
    ├── 地图 /map
    │   ├── 标记摘要（页面内浮层）
    │   └── 记录详情 /records/:recordId
    └── 我的 /profile
        ├── 数据统计 /statistics
        ├── 标签管理 /tags
        └── 设置 /settings
            ├── 缓存管理 /settings/cache
            ├── 隐私说明 /settings/privacy
            └── 关于 /settings/about
```

`/records/create` 与 `/records/:recordId/edit` 共用表单组件，但路由和提交语义保持独立。筛选在手机端可实现为底部抽屉；路由用于状态恢复和自动化测试，不要求表现为完整页面。

## 4. 核心用户流程

### 4.1 首次登录

1. 启动页检查本地 `refreshToken`。
2. 有有效会话时静默刷新并进入首页。
3. 无有效会话时进入登录页，说明微信授权用途与隐私边界。
4. 用户主动点击“微信登录”，客户端获取一次性授权码并交给后端。
5. 登录成功后拉取 `/me` 与首页摘要；失败时停留在登录页并提供重试。

不得在页面加载时自动弹出微信授权，不得把微信临时 `code` 写入业务日志或持久缓存。

### 4.2 从腾讯 POI 创建记录

1. 用户从首页或清单点击“添加店铺”。
2. 进入店铺搜索页，输入关键词，可选择城市、当前位置或地图中心点。
3. 选择一个 POI，进入新建记录页；店铺信息先作为未持久化草稿。
4. 用户至少选择 `status`，补充可选体验字段。
5. 提交时后端在一个事务内保存或复用店铺快照并创建记录。
6. 成功后清理草稿、刷新相关缓存并进入记录详情。

### 4.3 手动创建记录

1. 店铺搜索无结果时点击“手动添加”。
2. 输入 `name`，通过地图选点得到 `latitude`、`longitude`，地址和分类可补充。
3. 校验通过后进入新建记录页。
4. 提交过程与 POI 创建一致，不产生孤立的手动店铺。

### 4.4 打卡与状态切换

1. 用户可从“想去”详情点击“标记已打卡”。
2. 表单预选 `VISITED`，`visitedAt` 默认用户当地当天。
3. 用户可补充评分、消费、菜品、标签、备注和图片。
4. 保存成功后记录只出现在“已打卡”，首页、地图与统计同步失效并重新获取。
5. 任意状态切换为 `WANT_TO_GO` 且已有体验字段时，页面提示这些字段会保留，可选择“保留并切换”或返回手动清空。

### 4.5 删除记录

1. 详情页更多菜单中选择删除。
2. 对话框显示店铺名，并说明记录和关联图片将删除。
3. 用户二次确认后提交删除请求。
4. 成功后返回原清单或地图，清除详情缓存，相关统计刷新。
5. 失败时保留当前页面和内容，显示可重试错误，不做乐观删除。

## 5. 页面规格

### 5.1 登录页 `/login`

**内容**

- 产品名称与简短价值说明；
- 私人数据说明；
- “微信登录”主按钮；
- 隐私说明入口；
- 登录中、失败和重试状态。

**验收要点**

- 用户点击后才发起微信授权。
- 连续点击只发起一次请求。
- 登录成功前不进入受保护页面。

### 5.2 首页 `/home`

**内容顺序**

1. 问候与当前城市/定位状态；
2. “添加店铺”主操作；
3. 三种状态数量卡片；
4. 最近更新记录，最多 5 条；
5. 基础统计摘要入口。

**空状态**

首次使用时显示“记录第一家店”，直接进入店铺搜索；不展示虚构示例数据。

### 5.3 店铺搜索页 `/stores/search`

**内容**

- 搜索输入框与城市提示；
- “使用当前位置”和“在地图中选择”入口；
- POI 结果列表：`name`、`category`、`address`、`distanceMeters`；
- 最近使用的搜索词仅保存在本机；
- “没有找到？手动添加”入口。

**状态**

- 输入防抖 300 至 500 ms；少于 2 个字符时不自动请求。
- 定位拒绝时允许按城市搜索。
- 供应商超时、配额限制和无结果使用不同提示。
- 同一用户已记录的 POI 显示当前状态，点击优先进入已有详情，避免重复创建。

### 5.4 手动添加店铺页 `/stores/manual`

**字段**

| 字段 | 必填 | 交互 |
| --- | --- | --- |
| `name` | 是 | 文本输入，1 至 100 字符 |
| `category` | 否 | 文本输入或最近分类建议 |
| `address` | 否 | 地图选点后可编辑 |
| `latitude` / `longitude` | 是 | 通过地图选点，不要求用户直接输入 |
| `phone` | 否 | 文本输入，不自动拨号验证 |

离开页面时若有未保存修改，提示保留本地草稿或放弃。

### 5.5 新建/编辑记录页

Route：`/records/create`、`/records/:recordId/edit`。

**表单分组**

1. 店铺：名称、地址、位置，编辑时可展开修改个人快照；
2. 状态：三选一；
3. 用餐体验：`visitedAt`、四项评分、`perCapitaPrice`；
4. 菜品：推荐菜、踩雷菜；
5. 标签与备注；
6. 图片：选择、压缩、上传进度、排序、失败重试；
7. 保存按钮。

**交互规则**

- 选择 `WANT_TO_GO` 时收起体验区，但已填写内容不被自动删除。
- 每项评分支持 0.5 步长并可以清空。
- 图片在记录首次保存后上传；新建页先保存记录，再并发上传图片并显示整体完成状态。
- 表单变化后自动保存本地草稿；包含本地图片路径的草稿过期后提示重新选择。
- 编辑提交携带 `version`；收到 `409` 时展示“记录已在别处更新”，允许刷新，不自动覆盖。
- 保存进行中禁用重复提交，创建请求携带 `Idempotency-Key`。

### 5.6 清单页 `/records`

**结构**

- 顶部搜索框；
- 三个状态页签及数量；
- 筛选与排序按钮；
- 记录卡片列表；
- 浮动或导航栏“添加”入口。

**记录卡片最小信息**

- 店铺名、分类和区域；
- 状态；
- `overallRating`、`perCapitaPrice`、`visitedAt`（有值才显示）；
- 最多 3 个标签；
- 首图缩略图或稳定的无图占位；
- 距离（位置可用且请求了距离时）。

**列表行为**

- 使用游标分页，每页默认 20 条。
- 下拉刷新会清除当前查询的下一页游标。
- 搜索、筛选和排序变更时回到列表顶部。
- 空状态应复述生效的筛选条件，并提供“清除筛选”。

### 5.7 筛选 `/records/filter`

| 分组 | 控件 |
| --- | --- |
| `status` | 多选；从某状态页进入时默认锁定当前状态，可手动展开跨状态 |
| `tagIds` | 多选用户标签 |
| `ratingMin` / `ratingMax` | 评分范围 |
| `priceMin` / `priceMax` | 人均消费范围 |
| `visitedFrom` / `visitedTo` | 日期范围 |
| `sort` | `UPDATED_DESC`、`VISITED_DESC`、`RATING_DESC`、`PRICE_ASC`、`DISTANCE_ASC` |

距离排序缺少定位时禁用并解释原因。点击“应用”才改变列表，“重置”只清空抽屉中的条件，关闭不应用。

### 5.8 记录详情 `/records/:recordId`

**内容顺序**

1. 店铺名称、状态和编辑/删除菜单；
2. 地址、分类、电话、地图定位；
3. 图片画廊；
4. 四项评分与人均消费；
5. 推荐菜和踩雷菜；
6. 标签、到店日期和备注；
7. 创建与更新时间。

非当前用户的 `recordId` 与不存在资源使用同一“记录不存在或已删除”页面。签名图片 URL 失效时静默刷新一次，仍失败则显示占位与重试。

### 5.9 地图页 `/map`

**结构**

- 全屏腾讯地图；
- 状态筛选和图例；
- “定位到我”和“搜索此区域”控件；
- 地图底部的标记摘要卡；
- 新增记录入口。

**加载策略**

- 首次优先使用上次视口；有定位授权时可定位到用户。
- 只在用户点击“搜索此区域”或停止移动后明确触发加载，防止连续请求。
- 请求传递 `north`、`south`、`east`、`west` 和状态筛选。
- 返回超过上限时不随机截断，提示缩小地图范围。
- 聚合点是后续优化；MVP 可由客户端在标记密集时进行简单聚合，具体取决于地图 SDK 能力。

### 5.10 数据统计页 `/statistics`

**内容**

- 记录总数；
- 三种状态数量与占比；
- 平均总体评分及样本数；
- 平均人均消费及样本数；
- 最常用标签 Top 10。

当样本为空时显示“暂无数据”，不得以 0 分或 0 元代替未知值。MVP 不提供公开对比、年度海报或复杂趋势图。

### 5.11 标签管理页 `/tags`

- 展示标签名、颜色和关联记录数；
- 新建、改名、改色、删除；
- 标签重名使用已有标签并提示；
- 删除有关联记录的标签时二次确认，只解除关联，不删除记录。

### 5.12 设置页 `/settings`

MVP 设置项：

- 时区显示（默认 `Asia/Shanghai`，MVP 可只读跟随设备）；
- 缓存占用与“清除缓存”；
- 隐私说明；
- 应用版本、服务条款与第三方服务说明；
- 退出登录。

账号注销、自助导出等能力需在正式发布前结合实际上架地区要求评审；若进入发布范围，必须单独补充需求和验收用例。

## 6. 跨页面状态与缓存

### 6.1 状态分层

| 状态 | 归属 | 示例 |
| --- | --- | --- |
| 服务端状态 | API + PostgreSQL | 用户、店铺、记录、标签、图片元数据 |
| 会话状态 | 安全存储 + 内存 | `accessToken`、`refreshToken`、当前 `userId` |
| 查询缓存 | Riverpod/小程序数据层 + 本地缓存 | 列表、详情、统计、地图标记 |
| UI 状态 | 页面本地 | 当前页签、抽屉开关、地图视口 |
| 草稿状态 | 用户命名空间本地存储 | 未提交的记录表单与手动店铺 |

### 6.2 缓存键约定

缓存键必须包含 `userId` 和 `schemaVersion`，示例：

```text
foodtrace:v1:{userId}:record-list:{queryHash}
foodtrace:v1:{userId}:record-detail:{recordId}
foodtrace:v1:{userId}:record-draft:{draftId}
foodtrace:v1:{userId}:statistics:overview
```

写操作成功后的最小失效范围：当前记录详情、涉及状态的列表、首页摘要、地图标记和统计。退出登录时删除当前用户令牌、查询缓存和含私人数据的内存状态；草稿是否保留由退出确认框明确告知，默认删除。

## 7. 通用页面状态

每个含远程数据的页面必须设计以下状态：

- `initialLoading`：首次加载使用骨架屏或明确进度；
- `refreshing`：保留旧内容并显示刷新状态；
- `empty`：说明为空原因并给出一个主操作；
- `offline`：展示缓存时间，禁用需要联网的提交；
- `error`：提供可理解说明、`requestId` 和重试；
- `permissionDenied`：定位或相册权限被拒时解释替代路径；
- `submitting`：防止重复提交；
- `conflict`：并发版本冲突时不覆盖本地内容。

## 8. 组件规划

业务含义保持一致，组件分别用小程序自定义组件和 Flutter Widget 实现：

| 组件标识 | 用途 |
| --- | --- |
| `RecordCard` | 清单和最近记录摘要 |
| `StatusChip` | 三种状态展示与选择 |
| `RatingInput` | 0.5 步长评分输入 |
| `RatingSummary` | 四项评分只读展示 |
| `TagSelector` | 标签搜索、新建和多选 |
| `DishEditor` | 推荐菜/踩雷菜编辑 |
| `ImagePickerGrid` | 选择、排序、进度、重试和删除 |
| `StoreSummary` | 店铺名称、分类、地址与距离 |
| `FilterPanel` | 搜索筛选条件 |
| `EmptyState` | 空状态与主操作 |
| `ErrorState` | 错误、`requestId` 和重试 |
| `OfflineBanner` | 缓存时间与离线提示 |

组件不得直接持有永久密钥或绕过统一请求层访问后端。

## 9. 平台差异

| 能力 | 微信小程序 | Flutter App |
| --- | --- | --- |
| 微信登录 | `wx.login` | 微信开放平台移动应用授权 |
| 导航 | `tabBar` + 页面栈 | `NavigationBar` + 声明式路由 |
| 地图 | 小程序原生 `map` 组件 | 腾讯地图 Flutter/原生 SDK 适配层 |
| 安全存储 | 微信提供的本地能力，令牌最小化保存 | Keychain / Android Keystore 封装 |
| 图片选择 | `wx.chooseMedia` | 平台图片选择插件 |
| 网络层 | 统一封装 `wx.request` | Dio interceptor 与统一错误映射 |
| 状态管理 | 页面状态 + 独立数据仓库层 | Riverpod（MVP 首选） |

Flutter 选择 Riverpod，原因是依赖注入、异步状态与可测试性能够以较少模板代码覆盖本项目；若初始化阶段发现团队已有 Bloc 标准，再提交变更理由供审核。

## 10. 可访问性与视觉基线

- 状态使用图标、文字和颜色共同表达。
- 正文和辅助文字保持可读对比度，不把低对比灰字作为唯一信息来源。
- 评分控件提供文字数值，屏幕阅读器可识别当前分数。
- 图片必须有店铺名或序号形式的语义描述。
- 删除使用危险操作样式并二次确认；保存是页面唯一主按钮。
- 空状态和错误状态不使用会误导用户的虚构餐厅或统计数字。

## 11. 页面级验收清单

- 所有 Route 均有加载、空、错误和无权限路径。
- 从店铺搜索到新建成功不存在孤立店铺数据。
- 清单、详情、地图、首页和统计对同一次写入最终一致，并在前台主动失效缓存。
- Android 返回键、iOS 返回手势和小程序返回行为不会丢失未保存表单且无提示。
- 非本人或已删除详情不泄露资源是否曾存在。
- 长店铺名、长地址、9 张图片、10 个标签和 5,000 字备注下页面仍可操作。
- 定位、相册和微信授权拒绝均有可继续使用的替代路径或明确说明。
